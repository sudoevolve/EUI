# EUI-NEO 稳定版路线图

本文记录稳定版前的核心任务和当前进度。每一步保持 GLFW 默认路径可构建，并为新增后端单独验证。

## 总体顺序

1. 完善无边框自绘边框
2. 抽象成单头文件引入他人项目
3. SDL2 窗口支持
4. Vulkan 后端支持
5. 修复 bug 到稳定版

## 架构约束

稳定版前必须收束到清晰分层，避免继续用临时适配把 GLFW、SDL2、OpenGL、Vulkan 和平台原生能力缠在一起。

- 对外 app/DSL 层不得暴露 `GLFWwindow*`、`SDL_Window*`。内部统一使用 `core::window::Handle` 或后续 `WindowId`。
- 长期目标是单 `AppRunner`，CMake 只选择窗口后端和渲染后端；不长期维护两套重复主循环。
- `WindowBackend` 只负责窗口、事件、输入法、剪贴板、光标、图标、native handle 和父子窗口关系，不负责具体绘制。
- `RenderBackend` 单独负责 OpenGL/Vulkan surface、frame lifecycle、resize、present 和 primitive 渲染，不塞进 `WindowBackend`。
- GLFW/SDL2 事件必须先翻译为 EUI 自有事件，再进入 runtime；runtime/组件层不得出现后端键码和窗口库类型。
- 子窗口、modal、父子层级和多窗口更新归统一 WindowManager/AppRunner 管理，后端只提供底层窗口操作。
- macOS/Windows/Linux 原生能力集中到 native bridge，避免平台特例散落在窗口和渲染主循环里。
- Vulkan 开始前必须先完成窗口后端与渲染后端边界，不能把 Vulkan 特例散落在 GLFW/SDL2 main 里。
- 架构收紧时不得留下旧路径兼容壳或无用代码；迁移后要同步删除旧文件、旧 include、旧文档路径和未使用 CMake source。
- 每轮整理必须用 `rg` 扫旧路径/旧命名残留，并跑 `git diff --check` 与对应后端构建验证。

## 架构收束计划

1. 已完成：把 GLFW/SDL2 共享的 DSL 子窗口 compose/update/render 生命周期抽成公共类。
2. 已完成第一阶段：抽内部 `DslWindowManager`，统一 child window、modal、关闭、批量更新和动画状态查询。
3. 已完成第一阶段：抽内部 `AppRunner`，共享帧时间、render 标记、FPS 标题、tray 请求和 external-ready 分发；GLFW/SDL2 事件泵仍独立。
4. 已完成：示例源码迁移到 `examples/`，并建立 `include/eui_neo.h` 作为未来公共 facade 入口。
5. 已完成：抽 `MainWindowRuntime`，GLFW/SDL2 主窗口共用帧时间、external-ready、`update -> pending window hook -> render -> present`、子窗口更新回调、FPS 标题和 frame clock 流程；事件泵与窗口创建仍由后端独立实现。
6. 已完成前置接口：定义内部 `RenderBackend`，当前 OpenGL 主窗口 present 已走统一接口；Vulkan 接入前还需继续把 primitive 渲染从 OpenGL 细节中拆出。

## Core 分区计划

目标：逐步把 `core/` 收束成 `app / dsl / render / platform` 四类，不做一次性大搬家。

已完成：

- `RenderBackend` 已迁到 `core/render/render_backend.h`，不再放在 `core/app/`。
- 原生平台桥接已集中到 `core/platform/native_bridge.*`，覆盖 app icon、Dock icon 和 native child window 关系。
- `primitive.h`、`text.*`、`image.*` 已迁移到 `core/render/`，不保留旧路径兼容层。
- `async.*`、`network.*`、`event.h`、`platform.*`、`window_backend.*`、`tray_bridge.*`、`ime_bridge.*` 已迁移到 `core/platform/`，不保留旧路径兼容层。

下一步：

- 进入 Vulkan 前置：抽 primitive/text/image/render cache 的后端中立接口，先让 OpenGL 走新接口。

## 1. 完善无边框自绘边框

状态：暂缓。第一轮实现已撤销，后续应按平台原生窗口能力重新设计，不再用透明 framebuffer 模拟系统窗口外形。

## 2. 抽象成单头文件引入他人项目

状态：完成。

目标：提供稳定的单公开头 facade，配合 `eui::neo` CMake target 使用。当前依赖较多，不承诺真正纯 header-only。

已完成：

- `include/eui_neo.h` 已作为示例和外部 app 的单入口 facade，聚合 DSL app、组件、网络、平台和图片能力。
- `examples/*.cpp` 已统一只 include `eui_neo.h`，不直接拼 `components/*` 或 `core/*` 头文件。
- 已新增 `include/eui/types.h`、`network.h`、`platform.h`、`image.h`、`window.h` 薄 facade；`include/eui_neo.h` 不再直接 include `core/platform/*` 或 `core/render/*`。
- 已新增 `include/eui/async.h`，并把 `include/eui/dsl_app.h` 拆成公开声明头和 `include/eui/detail/dsl_app_impl.h` 实现头，重的 `glad/stb/runtime/text` 依赖集中到 detail。
- 已新增 `include/eui/dsl.h`，公开示例的页面入口签名迁到 `eui::Ui` / `eui::Screen`，hit-test 文档迁到 `eui::dsl::HitTestMode`。
- `eui::Color`、`eui::Shadow`、`eui::Transition`、`eui::Ease`、`eui::AnimProperty`、`eui::Align`、`eui::HorizontalAlign`、`eui::VerticalAlign`、`eui::SizeValue`、`eui::PointerEvent`、`eui::TextStyle`、`eui::mixColor(...)`、`eui::utf8(...)` 已覆盖示例常用基础类型和工具。
- 用户示例中的网络、平台和图片 ready 查询已改走 `eui::network`、`eui::platform`、`eui::ImagePrimitive`。
- `examples/*.cpp` 已不再出现 `core::...` 用户侧写法，保留底层实现和嵌入式自管 Runtime 文档中的 internal core 路径。
- 组件文档和内部开发 SKILL 已同步使用 `eui::Ui` / `eui::Screen`，避免继续传播旧 `core::dsl::*` 写法。
- README、中文 README 和集成指南已按三种方式重写：公共头文件方式、静态库方式、直接在 `examples/` 开发。
- 完整 app 文档示例已统一使用 `#include "eui_neo.h"`；嵌入已有 GLFW/OpenGL 主循环的文档仍允许直接使用 internal core 头。
- 已做外部项目 smoke test：临时项目只 `#include "eui_neo.h"`，通过 `add_subdirectory(...)`、`core/app/glfw_app_main.cpp` 和 `eui_neo_configure_app(...)` 构建 `smoke_app`。

后续维护约束：

- public facade 新增能力优先进入 `include/eui/*`，不要让 examples 重新直接依赖 `core/*` 路径或 `core::...` 用户侧命名。
- 后续 public API 改动必须同步更新 README、中文 README、集成指南和本路线图，避免文档漂移。

## 3. SDL2 窗口支持

状态：完成。

已完成：

- 增加 `EUI_WINDOW_BACKEND=glfw|sdl2` CMake 选项，默认仍为 GLFW。
- 增加 `core/platform/window_backend.*` 薄适配层。
- SDL2 路径已跑通 OpenGL context、DPI、鼠标、键盘、滚轮、剪贴板和光标。
- SDL2 IME rect 会从 framebuffer 像素换算回窗口逻辑坐标。
- SDL2 已加入 DSL 子窗口基础生命周期和 modal 输入路由。
- SDL2 不 vendored；默认 GLFW 构建不触发 SDL2，SDL2 后端使用系统包，或在 `EUI_DEPS_MODE=fetch` 时下载 SDL2 2.32.10。
- `/build-sdl2` 已加入 `.gitignore`。
- README、中文 README、开发文档、窗口文档和集成指南已补充 SDL2 构建命令。
- 已手动复测 SDL2 中文 IME 候选框定位。
- 已手动复测 SDL2 `app::openWindow(...)` modal 子窗口打开、交互和关闭。
- 已手动复测 SDL2 托盘、窗口焦点和多窗口边缘行为。

验证命令：

```sh
cmake -S . -B build-sdl2 -DCMAKE_BUILD_TYPE=Release -DEUI_WINDOW_BACKEND=sdl2
cmake --build build-sdl2 --target gallery --parallel
./build-sdl2/gallery

cmake -S . -B build-sdl2-fetch -DCMAKE_BUILD_TYPE=Release -DEUI_WINDOW_BACKEND=sdl2 -DEUI_DEPS_MODE=fetch
cmake --build build-sdl2-fetch --target gallery --parallel
```

## 4. Vulkan 后端支持

状态：前置接口进行中。

已完成：

- 增加内部 `core/render/render_backend.h`，定义 `RenderBackend` 和当前 `OpenGLRenderBackend`。
- 主窗口 `app::render(...)` 和 present 已通过 `MainWindowRuntime` + `RenderBackend` 串联，GLFW/SDL2 main 不再各自手写主窗口 update/render/swap 流程。

待继续：

- 抽 primitive/render cache 的后端接口；当前 `core::dsl::Runtime`、primitive、text、image 仍直接调用 OpenGL。
- 定义 Vulkan surface、frame lifecycle、resize、present 和资源释放策略。
- 保持 GLFW/SDL2 窗口后端只提供 native handle / surface 创建所需信息，不把 Vulkan 分支塞进事件泵。

## 5. 修复 bug 到稳定版

状态：持续进行。

要求：

- 每个阶段都回归 GLFW 默认构建。
- 发布前构建验证 `gallery`、`demo`、`clock`、`calculator`、`serial_tool`。
- 最后阶段冻结功能，只修复 bug、补文档和做发布检查。
