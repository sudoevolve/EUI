# 开发与发布

本文记录 EUI-NEO 的本地开发、依赖模式、构建验证和 release 约定。目标是让每次改动都能被快速复现、审查和发布。

## 依赖模式

项目通过 `EUI_DEPS_MODE` 控制构建期依赖来源：

- `auto`：默认模式。优先使用 `3rd/` 下的本地源码，缺失时才从固定上游地址联网拉取。
- `bundled`：严格离线模式。只使用 `3rd/` 下的源码，缺失依赖直接报错。
- `fetch`：强制联网模式。从固定上游地址拉取构建期依赖。

HarfBuzz shaping 默认启用，可用 `-DEUI_ENABLE_HARFBUZZ=OFF` 关闭。

## CMake Targets

项目导出 `eui::neo` 静态库 target。顶层构建时 `EUI_BUILD_APPS` 默认开启，会构建 `app/*.cpp` 下的示例程序；作为其他项目的子目录接入时，`EUI_BUILD_APPS` 默认关闭，只提供库和辅助函数。

- `eui::neo`：核心 DSL、Runtime、图元、组件可用的库 target。
- `eui_neo_configure_app(target)`：给完整 EUI app 配置平台链接选项并复制 `assets/`。
- `eui_neo_copy_assets(target)`：只复制 EUI-NEO 自带运行资源。

## 本地构建

macOS / Linux:

```sh
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel
./build/gallery
```

Windows / PowerShell:

```powershell
cmake -S . -B build
cmake --build build --config Release
.\build\Release\gallery.exe
```

Linux 依赖示例：

```sh
sudo apt-get install -y ninja-build libx11-dev libxrandr-dev libxinerama-dev libxcursor-dev libxi-dev libgl1-mesa-dev libcurl4-openssl-dev
```

## 验证命令

常规改动至少验证严格离线构建：

```sh
cmake -S . -B /tmp/eui-bundled -DCMAKE_BUILD_TYPE=Release -DEUI_DEPS_MODE=bundled
cmake --build /tmp/eui-bundled --parallel
```

依赖链路改动额外验证联网 fetch：

```sh
cmake -S . -B /tmp/eui-fetch -DCMAKE_BUILD_TYPE=Release -DEUI_DEPS_MODE=fetch
cmake --build /tmp/eui-fetch --parallel
```

发布前检查源码包体积：

```sh
git archive --format=zip --output=/tmp/eui-source.zip HEAD
ls -lh /tmp/eui-source.zip
```

## 提交规范

项目提交使用带 scope 的 Conventional Commits：

```text
type(scope): subject
```

允许的 type：

- `feat`：新增功能。
- `fix`：修复问题。
- `docs`：文档改动。
- `style`：格式、空白、代码风格，不改变行为。
- `refactor`：重构，不新增功能也不修复问题。
- `perf`：性能优化。
- `test`：测试相关。
- `build`：构建系统或依赖管理。
- `ci`：CI / release workflow。
- `chore`：维护性改动。
- `revert`：回滚提交。

规则：

- 必须包含 scope，例如 `docs(readme)`、`ci(release)`、`fix(input)`。
- type 和 scope 使用小写英文。
- subject 使用简短英文祈使句，不加句号。
- 避免 `update docs`、`fix bug`、`change files` 这类模糊描述。
- 提交前用 `git log --oneline -5` 确认新提交信息符合规范。

示例：

```text
docs(readme): align dependency setup documentation
ci(release): add macos release packaging
docs(workflow): add development and review guides
fix(input): correct emoji caret positioning
feat(text): add bundled emoji rendering
chore(deps): trim bundled third-party sources
```

## Release 约定

- 推送 `v*` tag 会触发 `.github/workflows/release.yml`。
- CI 构建 Windows、Linux、macOS 运行包。
- Release assets 只上传运行包，不上传正文截图。
- Release 正文截图使用已提交的仓库图片或 GitHub 正文附件。
- `docs/releases/` 是本地 release 草稿和素材目录，已被 `.gitignore` 忽略，不提交。

## Release 检查

发布前确认：

- `git status --short` 为空。
- `vX.Y.Z` tag 指向预期提交。
- Windows、Linux、macOS workflow 均通过。
- Release assets 只包含运行包。
- GitHub 自动 Source code 包体积处于合理范围。
- Release note 中的截图能在网页正文显示。
