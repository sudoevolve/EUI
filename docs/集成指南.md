# 集成指南

本文面向想把 EUI-NEO 接入自己 C++ 项目的用户。当前推荐两种方式：

- 用 `main.cpp` + `app/dsl_app.h` 快速创建完整桌面 app。
- 在已有 GLFW / OpenGL 主循环中只使用 `core::dsl::Runtime` 和组件层。

EUI-NEO 通过 CMake 导出一个库 target：

```cmake
eui::neo
```

当 EUI-NEO 作为子目录接入时，默认不会构建仓库自带的 `gallery`、`demo` 等示例程序。需要显式构建示例时，配置 `-DEUI_BUILD_APPS=ON`。

## 小白最快接入

最简单的方式是直接下载 EUI-NEO，放到自己项目里：

```text
MyProject/
  CMakeLists.txt
  app.cpp
  external/
    EUI-NEO/
```

`MyProject/CMakeLists.txt`：

```cmake
cmake_minimum_required(VERSION 3.14)
project(MyProject LANGUAGES C CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_subdirectory(external/EUI-NEO)

add_executable(my_app
    external/EUI-NEO/main.cpp
    app.cpp
)
eui_neo_configure_app(my_app)
```

`MyProject/app.cpp`：

```cpp
#include "app/dsl_app.h"
#include "components/components.h"

namespace app {

const DslAppConfig& dslAppConfig() {
    static const DslAppConfig config = DslAppConfig{}
        .title("My App")
        .pageId("my_app")
        .windowSize(900, 600);
    return config;
}

void compose(core::dsl::Ui& ui, const core::dsl::Screen& screen) {
    ui.column("root")
        .size(screen.width, screen.height)
        .padding(32.0f)
        .spacing(12.0f)
        .content([&] {
            ui.text("title")
                .text("Hello EUI-NEO")
                .fontSize(28.0f)
                .lineHeight(36.0f)
                .build();

            components::button(ui, "button")
                .text("Click me")
                .onClick([] {
                    // handle click
                })
                .build();
        })
        .build();
}

} // namespace app
```

构建：

```sh
cmake -S . -B build
cmake --build build
```

这条路径会让 EUI-NEO 接管窗口、事件循环和 OpenGL 渲染。你只需要实现 `app::dslAppConfig()` 和 `app::compose()`，然后在 `compose()` 里用 DSL 和 components 写界面。

## add_subdirectory

如果把 EUI-NEO 放在项目的 `external/EUI-NEO`：

```cmake
cmake_minimum_required(VERSION 3.14)
project(MyApp LANGUAGES C CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

add_subdirectory(external/EUI-NEO)

add_executable(my_app main.cpp app.cpp)
eui_neo_configure_app(my_app)
```

`eui_neo_configure_app(my_app)` 会链接 `eui::neo`，设置平台相关链接选项，并把 EUI-NEO 的 `assets/` 复制到可执行文件目录。这样默认字体、图标、图片和示例资源可以按 `assets/...` 路径找到。

如果你的 app 使用 EUI-NEO 自带的 `main.cpp` 和 `app/dsl_app.h`，写法类似：

```cmake
add_executable(my_app
    external/EUI-NEO/main.cpp
    app.cpp
)
eui_neo_configure_app(my_app)
```

如果你已经有自己的窗口系统和链接选项，只想使用库 target，也可以手动链接：

```cmake
target_link_libraries(my_app PRIVATE eui::neo)
eui_neo_copy_assets(my_app)
```

## FetchContent

也可以用 CMake `FetchContent`：

```cmake
include(FetchContent)

FetchContent_Declare(
    eui_neo
    GIT_REPOSITORY https://github.com/sudoevolve/EUI-NEO.git
    GIT_TAG        main
)
FetchContent_MakeAvailable(eui_neo)

add_executable(my_app main.cpp app.cpp)
eui_neo_configure_app(my_app)
```

实际项目建议把 `GIT_TAG` 固定到 release tag 或具体 commit，避免上游变化影响构建复现。

## 完整 App 模式

这是最容易开始的方式：让 EUI-NEO 管 GLFW 窗口、OpenGL context、事件循环、DPI、dirty rect 和多窗口请求。

`app.cpp`：

```cpp
#include "app/dsl_app.h"
#include "components/components.h"

namespace app {

const DslAppConfig& dslAppConfig() {
    static const DslAppConfig config = DslAppConfig{}
        .title("My EUI App")
        .pageId("my_app")
        .windowSize(900, 600)
        .fps(90.0);
    return config;
}

void compose(core::dsl::Ui& ui, const core::dsl::Screen& screen) {
    ui.column("root")
        .size(screen.width, screen.height)
        .padding(32.0f)
        .spacing(12.0f)
        .content([&] {
            ui.text("title")
                .text("Hello EUI-NEO")
                .fontSize(28.0f)
                .lineHeight(36.0f)
                .build();

            components::button(ui, "button")
                .text("Click")
                .onClick([] {
                    // update your app state here
                })
                .build();
        })
        .build();
}

} // namespace app
```

对应 CMake：

```cmake
add_executable(my_app
    external/EUI-NEO/main.cpp
    app.cpp
)
eui_neo_configure_app(my_app)
```

## 嵌入已有 GLFW / OpenGL 主循环

如果你的程序已经自己创建了 GLFW 窗口和 OpenGL context，可以直接持有一个 `core::dsl::Runtime`：

```cpp
#include "core/dsl_runtime.h"
#include "core/event.h"

core::dsl::Runtime runtime;
runtime.initialize(window);

while (!glfwWindowShouldClose(window)) {
    glfwPollEvents();

    int framebufferWidth = 0;
    int framebufferHeight = 0;
    glfwGetFramebufferSize(window, &framebufferWidth, &framebufferHeight);

    float contentScaleX = 1.0f;
    float contentScaleY = 1.0f;
    glfwGetWindowContentScale(window, &contentScaleX, &contentScaleY);
    const float dpiScale = (contentScaleX + contentScaleY) * 0.5f;
    const float logicalWidth = static_cast<float>(framebufferWidth) / dpiScale;
    const float logicalHeight = static_cast<float>(framebufferHeight) / dpiScale;

    runtime.compose("embedded", logicalWidth, logicalHeight, [](core::dsl::Ui& ui, const core::dsl::Screen& screen) {
        ui.row("root")
            .size(screen.width, screen.height)
            .content([&] {
                ui.text("hello")
                    .text("Embedded EUI-NEO")
                    .position(24.0f, 24.0f)
                    .fontSize(22.0f)
                    .build();
            })
            .build();
    });

    const bool shouldRender = runtime.update(window, deltaSeconds, dpiScale, dpiScale, true);
    if (shouldRender) {
        runtime.render(framebufferWidth, framebufferHeight, dpiScale);
        glfwSwapBuffers(window);
    }
}

runtime.shutdown();
```

嵌入模式下，你需要自己负责：

- `glfwInit()` / `glfwTerminate()`。
- 创建 OpenGL 3.3 core profile context。
- 调用 `gladLoadGLLoader(...)`。
- 安装或转发输入事件。直接使用 EUI-NEO 的 `core/event.h` 时，可调用 `core::installInputCallbacks(window)`。
- 主循环节流、清屏、swap buffers 和窗口生命周期。

如果不需要完全自定义主循环，优先使用完整 App 模式。

## 依赖和资源

EUI-NEO 的第三方依赖由 `3rd/dependencies.cmake` 处理：

- `EUI_DEPS_MODE=auto`：默认，优先用本地 `3rd/`，缺失时联网拉取。
- `EUI_DEPS_MODE=bundled`：严格离线，只使用 `3rd/`。
- `EUI_DEPS_MODE=fetch`：强制联网拉取。

HarfBuzz 默认开启，可通过 `-DEUI_ENABLE_HARFBUZZ=OFF` 关闭。

运行时资源默认按 `assets/...` 查找。外部项目可以：

- 调用 `eui_neo_copy_assets(my_app)` 复制 EUI-NEO 自带资源。
- 在自己的构建系统中复制资源。
- 通过 `DslAppConfig::textFont(...)`、`iconFont(...)`、`iconPath(...)` 指向自己的资源。

## 适合公开给用户的最小承诺

- C++17。
- CMake 3.14+。
- OpenGL 3.3 core profile。
- GLFW 窗口输入。
- `eui::neo` 是库 target；示例程序是可选目标。
